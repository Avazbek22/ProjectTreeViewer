## 1) Что это и для чего
ProjectTreeViewer быстро формирует контекст проекта: показывает структуру папок/файлов, позволяет выбрать элементы галочками и скопировать дерево и/или содержимое файлов в буфер обмена. Удобно для code review, поддержки, обучения и передачи структуры проекта AI-ассистенту.

Важно: приложение работает только в режиме чтения и не изменяет файлы.

## 2) Дерево проекта
* Иконки типов файлов/папок.
* Чекбоксы на узлах.
* Раскрытие/сворачивание.

Чекбоксы: отметка узла применяется к потомкам; состояние родителей пересчитывается автоматически.

Горячие клавиши:
* Ctrl+E — раскрыть всё
* Ctrl+W — свернуть всё (корень остаётся раскрытым)

## 3) File (Файл)
* Open folder… — открыть корневую папку (Ctrl+O)
* Refresh — перечитать и перестроить дерево (F5)

## 4) Copy (Копирование)
### Copy full tree
Копирует полное ASCII-дерево проекта. Ctrl+Shift+C.

### Copy selected tree
Копирует дерево по выбранным (отмеченным) узлам. Если ничего не выбрано — выводится сообщение. Ctrl+Alt+C.

### Copy selected content
Копирует содержимое только выбранных текстовых файлов:
* файл должен существовать;
* бинарные файлы пропускаются (проверка на нулевые байты);
* вывод сортируется и разделяется пустыми строками.

Ctrl+Alt+V.

### Copy tree + content
Комбинирует дерево и содержимое:
* дерево строится по выделению; если выделение невалидно — берётся полное дерево;
* контент добавляется только для текстовых файлов;
* если контента нет — копируется только дерево.

Ctrl+Shift+V.

## 5) View (Вид)
### Масштаб (Zoom)
* Ctrl++ / Ctrl+= — увеличить
* Ctrl+- — уменьшить
* Ctrl+0 — сброс
* Ctrl + колесо мыши над деревом — плавно

Диапазон: 6–28, сброс — 12.

### Compact Mode
Делает интерфейс плотнее: меньше отступы в дереве и настройках.

## 6) Search (Поиск по дереву)
Открывает панель поиска, подсвечивает совпадения и позволяет навигацию.
* Ctrl+F — показать/скрыть
* Enter — следующий результат
* Shift+Enter — предыдущий
* Esc — закрыть

Поиск автоматически раскрывает только нужные ветки; при очистке запроса дерево сворачивается (кроме корня).

## 7) Filter (Фильтр по имени)
Показывает только узлы, соответствующие строке фильтра, и их родителей. Обновление — с задержкой (debounce).
* Ctrl+Shift+N — показать/скрыть фильтр
* Esc — закрыть фильтр

## 8) Options (Панель настроек)
Справа доступны списки:
* Font (шрифт дерева) — применяется кнопкой Apply
* Ignore options
* Extensions
* Root folders

У каждого списка есть переключатель All (включить/выключить всё).

Ctrl+P — показать/скрыть панель.

## 9) Игнорирование и Smart Ignore
### Обычные правила
Можно исключать типичные артефакты и «шум» (например bin/obj, скрытые элементы, элементы с точкой и т.п.).

### Smart Ignore
Дополнительно применяются «умные» исключения (например VCS/IDE-папки и типичные мусорные файлы). Для фронтенд-артефактов правила активируются только если в проекте обнаружены соответствующие признаки (package/lock-файлы).

## 10) Как строится дерево
На результат влияют:
* выбранные Root folders
* выбранные Extensions
* Ignore rules
* активный Filter

Если нет разрешения на доступ к папке, узел помечается как Access denied.

## 11) Иконки
Иконка выбирается по:
1) типу узла (папка/файл)
2) точному имени (если есть правило)
3) расширению
4) иначе — unknown

## 12) Theme (Тема и эффекты)
Popover темы позволяет переключать:
* светлая/тёмная тема
* Transparent / Mica / Acrylic
* параметры интенсивности, размытия, контраста и границ

Пункты с пометкой [Beta] могут менять поведение между версиями.

## 13) Help → About (О программе)
Открывает popover с информацией о приложении и ссылкой на репозиторий.
* Esc — закрыть popover

## 14) Языки
Выбор языка выполняется через меню Language.

## 15) Примечания
* По умолчанию применяется тёмная тема и режим с эффектом прозрачности (если поддерживается системой).
* При нехватке прав приложение может предложить перезапуск с повышенными правами; при отказе покажет уведомление.
